\# Bank Transaction Fraud Detection Dashboard



\## 1. Project Overview



This project utilizes a bank transaction dataset to identify patterns of fraudulent activities. It focuses on building a data-driven Power BI dashboard that empowers analysts to detect fraud effectively using interactive visualizations and DAX calculations.



\---



\## 2. Objectives



\- Detect fraudulent transactions using transaction and balance patterns.

\- Visualize key fraud indicators through Power BI.

\- Provide actionable insights using DAX measures and filters.

\- Improve fraud monitoring and decision-making processes.



\---



\## 3. Dataset Summary



\### Dataset Information

\- Dataset Name: Bank Transaction Fraud Detection

\- Source: Simulated Banking Transactions

\- File Format: Excel (.xlsx)

\- Records: 100,000+ transactions

\- Fields:

&#x20; - Customer\_ID

&#x20; - Customer\_Name

&#x20; - Age

&#x20; - Gender

&#x20; - State

&#x20; - City

&#x20; - Account\_Type

&#x20; - Account\_Balance

&#x20; - Transaction\_ID

&#x20; - Transaction\_Amount

&#x20; - Transaction\_Date

&#x20; - Merchant\_Category

&#x20; - Device\_Type

&#x20; - Is\_Fraud



\---



\## 4. Dashboard Features



\### KPIs

\- Total Transactions

\- Total Transaction Amount

\- Fraud Transactions

\- Fraud Percentage

\- Average Transaction Value



\### Visualizations

\- Fraud by State

\- Fraud by Merchant Category

\- Fraud by Gender

\- Fraud by Device Type

\- Monthly Fraud Trend

\- Transaction Amount Distribution



\---



\## 5. Technologies Used



\- Microsoft Power BI

\- DAX (Data Analysis Expressions)

\- Power Query

\- Microsoft Excel



\---



\## 6. Key Problems \& Solutions



| Problem | Solution |

|----------|----------|

| Large transaction volume | Interactive filtering and aggregation |

| Fraud detection complexity | Fraud-specific KPIs and measures |

| Regional fraud analysis | State-wise visualization |

| Trend identification | Time-series analysis |



\---



\## 7. Insights Generated



\- States with the highest fraud occurrence.

\- High-risk merchant categories.

\- Fraud distribution across age groups and genders.

\- Device types commonly associated with fraudulent activities.

\- Monthly fraud trends and transaction behavior.



\---



\## 8. Conclusion



This Power BI dashboard serves as an effective fraud monitoring system by providing real-time insights into transaction patterns, accounts at risk, and potential fraudulent activities, helping organizations minimize fraud-related losses.
